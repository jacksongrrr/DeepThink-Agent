# 产品愿景

## 项目简介

**DeepThink Agent** 是本地运行的研究型 Web 应用：在生成「思考路径」之前，先用独立的 **问题分类 Chat** 输出**抽象画像**（领域类型、难度、问题结构、思考姿态等），**禁止在分类结果中复述用户原文敏感细节**，供下游路径模型**定调**；路径模型在「画像 + 原问题」下生成 **6～12 条过程性、可自检** 的思考步骤（优先「先…再…」「是否…？」），避免平庸的「xxx 分析」式名词堆砌。随后对每条路径 **并行调用 R1**，再经 **Chat 综合** 为单一回答。**对比模式**下，单次纯 R1 与技术管线流式**并行**。界面默认**仅突出结果**（基线回答、各路径小结、综合回答），分类、路径规划与 R1 CoT 置于**可折叠区域**。

## 目标用户

需要对照「分类约束下的多路径并行 R1 + 综合」与「单次 R1」的提示词研究者、开发者与产品经理。

## 核心场景

1. 用户输入开放问题，选择：仅基线、仅技术、或对比。
2. 技术侧：分类 → 路径生成 → 并行分路径 R1 → 综合；流式 NDJSON 推送。
3. 默认阅读：直接看回答与小结；需要审计时再展开分类、路径与推理过程。

## 成功标准

- `.env` 配置后一条命令可跑通端到端。
- 分类—路径—推理—综合链路清晰、可折叠 UI 不干扰主阅读路径。
- 提示词角色分离，Chat JSON 可解析，便于迭代。

## 非目标

- 无多租户鉴权、持久会话与生产部署方案。
- 不保证模型事实正确；聚焦流程与可观测性。
